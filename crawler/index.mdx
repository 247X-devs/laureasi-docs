Documentazione centralizzata delle Edge Functions presenti nel repository `laureasi-crawler`.

## Standard condiviso

- Metodo HTTP standard: `POST`
- Parametri standard: `snake_case`
- Retrocompatibilità: parametri `camelCase` accettati con warning in `meta.warnings`
- Envelope risposta:
  - Successo: `success: true`, payload in `data`, metadata in `meta`
  - Errore: `success: false`, dettagli in `error.code`, `error.message`, `error.details`

## Runbook data source

1. Eseguire il check di integrità dal control plane Payload
2. `reconcile-data-sources` in `dry_run=true`
3. `reconcile-data-sources` con `dry_run=false` dopo verifica
4. Scrape mirato delle righe in `unmatched_for_scrape`
5. `cleanup-entities` per duplicati/orfani

## Runbook update campi

1. `refresh-fields` con `mode: "scan"`
2. Verificare `data.report` e `summary.with_changes`
3. Se `with_changes > 0`, `mode: "apply"` o `refresh_auto_apply: true` dal workflow admin

## Deploy

```bash
supabase functions deploy --no-verify-jwt
```

Vedi anche [Edge Functions + Payload CMS](/piattaforma/edge-functions) per l'orchestrazione dal pannello admin.
