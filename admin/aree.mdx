Le **Aree Tematiche** raggruppano i percorsi per macro-settore (es. Economia, Informatica, Sanità).

## Dove trovarle

Menu laterale → **📝 Contenuti** → **Aree Tematiche**

  <img src="/images/admin/aree-lista.png" alt="Lista delle aree tematiche nel pannello admin" />
</Frame>

## Creare un'area

<Steps>
  <Step title="Crea nuovo">
    Clicca **Crea nuovo** nella lista aree.
  </Step>
  <Step title="Compila i campi">
    - **Titolo** — nome dell'area mostrato sul sito
    - **Slug** — parte dell'URL (es. `informatica`)
    - **Descrizione** — testo introduttivo
    - **Immagine rappresentativa** — icona o foto dell'area
    - **Macro area** — categoria di appartenenza (se prevista)
    - **In evidenza** — per mostrarla in homepage
  </Step>
  <Step title="Salva">
    L'area sarà raggiungibile su `/area/[slug]` sul sito pubblico.
  </Step>
</Steps>

## Collegamento ai percorsi

Ogni corso, laurea o master è associato a **una sola** area tematica (campo relazione nel CMS). Questo alimenta filtri, navigazione, Confronta-corsi e il quiz di orientamento.

## Consolidare o rinominare aree

Non modificare a mano in produzione le merge tra aree (remap di `area_id` + delete). Usa una **migrazione Payload** in `laureasi-admin`:

```bash
npm run migrate:create -- --name consolidate_areas_…
```

Nella migrazione, risolvi le aree per **slug** (gli id possono cambiare tra ambienti), rimappa `degrees` / `masters` / `courses`, poi elimina l'area di origine. Aggiungi redirect permanenti in `laureasi-frontend/vercel.json` per gli slug dismessi (`/area/…`, `/master/area/…`).

Esempio già applicato: Pedagogia → Scienze della Formazione e Pedagogia (`scienze-della-formazione`).

## Chip Confronta corsi

Nella pagina **Confronta corsi**, i chip “Esplora per area” possono aggregare **più aree CMS** o un’intera **macro area** (es. Economia → tutte le aree `economia_e_management`; Scienze Politiche → Politiche + Relazioni internazionali + Amministrazione pubblica).

Questo allinea il linguaggio prodotto al catalogo senza fondere le aree nel CMS: la tassonomia resta fine per nav e filtri.
